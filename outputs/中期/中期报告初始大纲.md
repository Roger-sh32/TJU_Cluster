## 二、研究背景与意义

随着自动驾驶技术的发展，车辆道路测试和仿真测试过程中产生的数据规模不断扩大。大量路测数据中包含丰富的交通参与者行为、道路结构、车辆运动状态和潜在风险信息，但其中真正具有复盘价值和测试价值的关键场景通常只占较小比例。如果完全依赖人工回看和规则筛选，不仅效率较低，而且难以覆盖未知的复杂场景和低频长尾场景。

自动驾驶测试中的关键场景往往具有低频、复杂交互、高风险或异常行为等特征。如何从大规模、无标注的路测数据中自动发现具有代表性和安全分析价值的场景，是提升测试效率和优化测试资源分配的重要问题。

本研究拟利用无监督聚类方法对自动驾驶路测场景进行结构化分析，通过构建场景级特征、识别场景类型，并结合稀有性、异常性和风险性进行关键场景排序，为自动驾驶测试数据复盘、场景库建设和测试优先级管理提供方法支持。

## 三、研究目标

本研究的总体目标是构建一种基于聚类的自动驾驶路测关键场景识别方法，实现从原始路测数据到高价值场景筛选结果的自动化分析流程。

具体目标包括：

1. 构建面向自动驾驶路测数据的场景级特征表示方法。
2. 利用无监督聚类方法识别不同类型的驾驶场景。
3. 设计关键场景综合评分机制，实现对高价值场景的排序与筛选。
4. 基于 nuScenes 数据集验证方法的可行性。
5. 为自动驾驶路测复盘、测试用例选择和场景库建设提供可解释的分析依据。

## 四、已完成的研究工作

### 4.1 文献调研与研究方案设计

已完成自动驾驶测试评价、场景化测试、场景表示、无监督聚类、异常检测和关键场景识别等方向的初步文献调研，并在开题阶段形成了研究问题、技术路线和论文结构设计。

近期已对开题报告中的参考文献进行真实性核验和初步整理，保留了与当前研究主线高度相关的中文综述文献，并补充了 nuScenes、UMAP、HDBSCAN、关键场景识别与安全关键场景综述等外文文献，为后续中期报告和论文写作提供参考基础。

### 4.2 数据处理环境搭建

已完成基于 nuScenes v1.0-trainval 数据集的本地实验环境搭建，能够读取 nuScenes 元数据、场景信息、车辆状态、交通参与者信息和地图相关信息，为场景级特征提取提供数据基础。

### 4.3 场景级特征构建

当前以 nuScenes 中的 scene 作为基本分析单元，对每个场景内的帧级信息进行统计聚合，形成场景级特征向量。

已构建的特征主要包括：

1. 自车运动特征：如速度、加速度、航向变化等统计指标。
2. 交通环境特征：如周围交通参与者数量、车辆密度和参与者结构。
3. 风险相关特征：如基于相对距离和相对运动状态的潜在风险指标。
4. 地图语义特征：如交叉口、人行横道等道路环境语义信息。

当前已完成 850 个 scene 的场景级特征提取，形成 `tables/scene_features.csv`。

### 4.4 UMAP + HDBSCAN 聚类实验

在完成场景特征构建后，已对特征进行标准化处理，并采用 UMAP 方法进行非线性降维，以降低高维特征空间的复杂度并保留局部相似结构。

随后采用 HDBSCAN 方法进行密度聚类。相比需要预先指定聚类数量的 KMeans 等方法，HDBSCAN 能够根据数据密度结构自动发现不同场景簇，并识别噪声样本，更适合处理自动驾驶场景数据中可能存在的非均匀分布、长尾分布和低频异常场景。

当前聚类实验输出包括：

- `tables/scene_features_umap_hdbscan.csv`
- `pictures/Figure_1.png`

### 4.5 关键场景综合评分

在聚类结果基础上，已初步构建关键场景综合评分方法。当前评分主要考虑以下三个维度：

1. 场景稀有性：根据场景所属簇的规模衡量该类场景在整体数据中的出现频率。
2. 簇内异常程度：衡量单个场景相对于所属簇内典型样本的偏离程度。
3. 风险特征水平：结合与安全风险相关的场景特征，反映场景潜在风险程度。

通过上述维度的综合计算，已生成包含 `rarity`、`atypical`、`risk`、`corner_score` 和 `corner_score_norm` 等字段的关键场景评分结果。

当前输出文件为：

- `tables/scene_features_with_corner_score.csv`
- `top20_corner_cases_analysis.xlsx`

## 五、阶段性实验结果

当前阶段已完成基于 nuScenes 数据集的初步实验验证，主要结果如下：

| 项目 | 当前结果 |
| --- | --- |
| 数据集 | nuScenes v1.0-trainval |
| 分析单元 | scene |
| 场景数量 | 850 |
| 场景特征数量 | 24 个数值型特征 |
| 聚类方法 | UMAP + HDBSCAN |
| 聚类标签数量 | 18 个，包含噪声类 |
| 有效簇数量 | 17 个 |
| 噪声样本占比 | 约 6.47% |
| 关键场景评分 | 已完成初步计算 |
| Top20 场景分析 | 已形成初步分析表 |

从当前结果看，UMAP + HDBSCAN 方法能够在无人工标注条件下对 850 个自动驾驶场景进行分组，并识别出一定比例的噪声或低频样本。结合关键场景评分后，可以对场景进行优先级排序，为后续人工回看和测试复盘提供候选列表。

当前实验说明，本研究提出的“场景级特征构建—无监督聚类—关键场景评分”技术路线具备初步可行性。
